# Custom REST API #

This plugin provides a controlled set of REST endpoints so that external systems (ERP, mobile apps, etc.) can access Moodle data in real time.
Unlike Moodle's native web services, only the strictly necessary information is exposed here, in a consistent and validated format.

## Features ##

1. Active Courses
    - Endpoint: /local/custom_restapi/courses
    - Returns a list of active courses with the number of enrolled students.
2. Student Progress in Course
    - Endpoint: /local/custom_restapi/progress
    - Requires parameters: userid, courseid.
    - Returns the student's progress percentage in the course.
3. Student Grades
    - Endpoint: /local/custom_restapi/grades
    - Requires parameter: userid.
    - Returns the student's grades for all their courses.
4. Autenticación y Tokens
    - Tokens are generated by the administrator in Moodle.
    - Tokens can be enabled or disabled.
    - If a token does not exist or is disabled, the API returns an HTTP 401 Unauthorized response.

## Installing via uploaded ZIP file ##

1. Log in to your Moodle site as an admin and go to _Site administration >
   Plugins > Install plugins_.
2. Upload the ZIP file with the plugin code. You should only be prompted to add
   extra details if your plugin type is not automatically detected.
3. Check the plugin validation report and finish the installation.

## Installing manually ##

The plugin can be also installed by putting the contents of this directory to

    {your/moodle/dirroot}/local/custom_restapi

Afterwards, log in to your Moodle site as an admin and go to _Site administration >
Notifications_ to complete the installation.

Alternatively, you can run

    $ php admin/cli/upgrade.php

to complete the installation from the command line.
